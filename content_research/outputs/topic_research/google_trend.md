# Content Research

## Research 목적 정의

```
- 어떤 산업의 어떤 담당자가 어떤 문제 때문에 어떤 키워드를 검색하고, 그 사람에게 어떤 콘텐츠를 보여줘야 리드로 전환될 가능성이 높은가?
```

## Q1-a. 담당자는 검색창에 뭘 칠까?

```
- CS서비스 구매 의도가 있는 사람이 검색할 법한 키워드
```

### 데이터 소스

```
- 네이버 검색광고 API - 일반인 x
- Google Ads API - 공개 x
- 네이버 데이터랩 API
- Google Trends API - 공개 x
```

## Q1-b. CS서비스를 필요로 하는 중소기업은 어떤 산업일까?

```
- CS 운영 부담이 큰 산업군 식별 
- 채용공고는 회사가 고객상담 인력을 필요로 한다는 신호와 가까움
```

### 데이터 소스

```
- 사람인 API - 기다리는 중
- 고용24 API - 개인에게 공개 x
```

## Q2. 특정 산업 담당자가 볼 만한 insight/콘텐츠 소재는 뭘까?

```
- 콘텐츠에 쓸 수 있는 문제 구조를 뽑는 것
```

### 데이터 소스

```
- 네이버 뉴스 검색 API
- 빅카인즈 (유료화)
```

## Workflow

### 1. Search Intent Discovery

```
- 담당자가 검색하는 키워드 수요/트렌드 분석
```

### 2. Industry Demand Detection

- CS 채용공고 기반 수요 산업 식별

### 3. Industry Issue Mining

- 뉴스 기반 산업별 CS 문제/트리거 수집

### 4. LLM Source Review

- 콘텐츠에 쓸 근거자료 선별 및 신뢰도 평가

### 5. Content Brief Generation

- 산업별·페르소나별 콘텐츠 기획서 생성

### 6. Multi-channel Outline Generation

- 블로그/매거진/링크드인/메타 소재로 변환

## 추후: LLM API 활용

```
- TF-IDF 로는 구매의도, 산업 적합도, 리드 전환 가능성, 콘텐츠 소재성 판단 어려움
```

### 1. Query expansion

```
- seed keyword 에서 산업별 검색어로 확장 (가지내리기)
```

### 2. Intent classification

```
- 키워드별 검색 의도 분류 (이건 보류)
```

### 3. LLM Source selection optimization

```
- 뉴스/채용/검색결과 데이터 기반 컨텐츠에 쓸 말한 source 골라주는 것
```

### 4. Content Brief 생성

```
- outline generator 까지 연결
```

## 가설산업 특정:

```
- 이커머스/쇼핑몰
- 렌탈/구독/AS
    - AS, 해지, 요금, 계약 문의가 반복 발생 --> 외주화/AI 상담/VOC 분석 연결
- 병원/의원
    - 예약 문의, 전화 폭주, 민원 응대가 반복 “예약 문의/일반 문의 응대 구조”로 접근
- 교육/학원
    - 상담 예약·환불 문의가 많은 교육업 CS 운영법
- 물류/배송
    - 쇼핑몰의 배송 문의 대응
```
