Goal

목표는 “완전 자동 작성”이 아니라 “팀원이 실제로 쓸 수 있는 콘텐츠 보조 웹앱”

1

입력 구조화

주제, 타겟 독자, 키워드, 연결할 솔루션, 소목차 방향을 form으로 받는다.

2

AI 초안 생성

기존 CS쉐어링 콘텐츠 흐름을 참고해 outline과 draft를 생성한다.

3

사람 검수

팀원이 outline과 draft를 직접 수정하고, 필요한 경우에만 reviewer를 실행한다.

핵심 방향: AI가 결과물을 “대신 완성”하는 게 아니라, 사람이 더 빠르게 콘텐츠를 만들 수 있도록 구조와 초안을 제공한다.
Why not Streamlit?

팀 배포 목적이면 Streamlit보다 FastAPI + HTML/JS가 더 적합

구분StreamlitFastAPI + HTML/JS
장점Python만으로 빠르게 MVP 가능진짜 웹페이지처럼 구성 가능, UI 자유도 높음
단점서비스 화면 느낌 약함, UI 커스터마이징 제한초기 세팅이 조금 더 필요
추천 용도혼자 테스트, 내부 데모팀원에게 링크로 배포, 장기 확장
현재 목표가 “팀원들이 들어가서 사용하는 앱”이면 FastAPI + HTML/CSS/JavaScript로 가는 것이 더 자연스럽다.

LLM Pipeline

Agent보다 먼저 “Prompt Function” 3개로 시작

1

Outline Planner

입력 form을 정리하고 콘텐츠 타입에 맞는 소목차 흐름을 만든다.

2

Draft Writer

사람이 수정한 outline을 기준으로 본문을 생성한다.

3

Flow Reviewer

중복, 비약, 어색한 CTA, 솔루션 연결의 과한 광고성을 점검한다.

처음부터 섹션마다 agent를 쪼개면 비용과 문체 흔들림이 커진다. MVP는 3개의 prompt function으로 충분하다.
Cost Control

비용을 줄이는 핵심은 “사람이 직접 수정하는 구간”을 넣는 것

단계API 호출비용 관리 방식
Brief 입력없음사용자 입력만 저장
Outline 생성1회저렴한 모델 사용 가능
Outline 수정없음textarea에서 사람이 직접 수정
Draft 생성1회좋은 모델 사용
Review선택필요할 때만 실행
MVP 최소 비용 버전은 콘텐츠 1개당 API 호출 2회: Outline 1회 + Draft 1회.
Output Format

AI 생성 원본은 HTML보다 JSON Blocks가 안전

{ "title": "성수기 CS대응, 인력보다 중요한 것은 운영 구조입니다", "category": "인사이트", "keywords": ["성수기 CS", "문의 폭증", "운영 구조"], "blocks": [ { "type": "heading", "level": 2, "text": "성수기 CS대응이란 무엇인가" }, { "type": "paragraph", "text": "성수기 CS대응은..." }, { "type": "table", "headers": ["구분", "고정형", "가변형"], "rows": [...] }, { "type": "faq", "question": "상담사를 더 뽑으면 되나요?", "answer": "..." } ] }
JSON blocks를 기준으로 홈페이지 HTML, 네이버 블로그용 복붙 원고, Markdown, 미리보기 HTML로 변환한다.
Export Strategy

플랫폼별 Export를 분리해야 한다

홈페이지 매거진

h2/h3, p, ul/li, table, FAQ, 이미지 위치를 HTML로 변환한다.

네이버 블로그

복잡한 HTML보다 에디터에 붙여넣기 쉬운 텍스트/표/이미지 위치 안내가 현실적이다.

검수용 Preview

팀원이 브라우저에서 읽고 수정할 수 있는 미리보기 HTML을 제공한다.

AI의 기본 출력은 JSON/Markdown, HTML은 최종 변환 옵션으로 두는 게 안전하다.

Risks

주의할 리스크와 대응

비용 증가

전체 재생성 버튼 남발, reviewer 자동 실행이 비용을 키울 수 있다.

대응: 사람 직접 수정 우선, 부분 재생성 버튼, 비용 경고 표시.

문체 흔들림

섹션별로 별도 호출하면 글 전체 톤과 흐름이 끊길 수 있다.

대응: 전체 outline 기반 draft 생성 + 마지막 reviewer 사용.

플랫폼 포맷 차이

홈페이지와 네이버 블로그는 업로드 방식이 다를 수 있다.

대응: JSON blocks를 원본으로 두고 export만 분리.

과한 자동화

초기부터 LangGraph/RAG/React까지 넣으면 일정이 커진다.

대응: FastAPI + HTML/JS MVP부터 시작.
